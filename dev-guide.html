마크업 관련 가이드

a. 마크업 파일의 title 태그 이름은 파일 명과 동일하게 구성

b. Html 내의 lang 값은 ko로 고정.

<html lang="ko">를 모든 마크업 파일 상단에 명기하여야 함.

c. Viewport 초기화 메타 태그는 모든 마크업 파일 내에 포함하여야 함.

<meta name="viewport" content="width=device-width,initial-scale=1,viewport-fit=cover,maximum-scale=1,user-scalable=no"/>

d. 마크업 작성 시 컴포넌트 용도에 맞는 태그를 사용.
Ex) div 태그의 스타일 속성을 임의로 구성하여 버튼 컴포넌트를 작성하는 등의 행위를 금함.

e. Button 태그 안은 span 태그만 위치할 수 있도록 마크업 구성, button 태그 하위 - 앵커 태그 구조로 구성되지 않도록 하여야 함.

f. 마크업 내의 class 값을 선언할 때에는 피그마 내의 컴포넌트 id와 1대 1로 대조될 수 있도록 구성. 임의의 네이밍 규칙 적용하여 새 클래스 명을 적용하지 않도록 하여야 함.

g. 피그마 파일 내의 text 섹션에 별도의 줄바꿈이 없다면, 강제 줄바꿈이 없는 것임. 페이지 내의 모든 내용은 피그마 파일 내의 내용을 기준으로 동일하게 구현되어야 함.

h. Input 태그의 경우 작성 시 용도에 맞는 type 값을 명기하여야 함.

i. 화면 상의 요소를 구현하기 위해 한 마크업 파일 내 h1 태그를 다중으로 사용하는 행위는 지양. Heading 태그 사용 시에는 h2 태그를 이용하여 처리.

스타일 관련 가이드

a. vw/vh 사용은 지양, viewport 화면 내의 비율로 요소 사이즈가 구성 되어야 할 때는 퍼센트 수치값을 사용.

b. Margin 속성을 이용한 간격 값 지정은 최대한 지양, 요소 간의 간격을 선언 할 때에는 padding을 사용해서 조정

c. 페이지 줌을 방지하기 위해 html {touch-action: pan-y} 값을 공통 스타일로 적용

d. 부모 요소가 fixed로 고정되어야 할 경우 (Header, Tabbar, Bottom-fixed Button 등의 컴포넌트), position 값은 부모 요소에 무조건 위치하여야 함. 자식 요소가 부모 요소를 건너뛰고 position 값을 갖지 않도록 구성

e. 상위 요소의 display 값이 flex를 가질 경우, 자식 요소 간의 간격을 지정할 때에는 margin, padding 등의 요소별 간격을 사용하는 것이 아닌 gap 속성을 사용하여 지정하여야 함.

f. 텍스트 컨텐츠를 포함하고 있는 요소의 경우 최대 범위만 잡아두고, width에 대한 고정 px는 사용하지 않도록 구성.

g. img 태그의 경우 div나 span으로 감싼 뒤, 상위 요소에 width: 100%; 속성 적용하여 채울 수 있도록 구성.

h. 이외 border-radius 속성 등 이미지를 감싸는 컨테이너에 속성값이 주어져야 하는 경우에도 마찬가지로, img 태그가 아닌 상위 요소에 해당 속성을 부여하여야 함.

i. 화면의 특정 영역에 고정되어 보여져야 하는 요소의 경우 position: absolute; 가 아닌 position: fixed; 를 사용하여 구성.

j. Select 태그의 경우 기본적으로 outline을 제거하고, 필요 시 outline 속성을 수정하여 사용할 수 있도록 하여야 함.

k. Checked 속성을 가지는 컴포넌트의 경우 별도의 클래스를 생성해서 관리하는 것이 아닌, css 내의 :checked 셀렉터를 이용하여 구현.

l. Input 태그 등의 글자 입력 이벤트는 js를 사용한 훅을 별도로 구성해서 관리하는 것이 아닌, css 내의 :placeholder-shown 셀렉터를 이용하여 구현.

m. 피그마 파일 기준 radius가 있는 컨테이너의 경우 border-radius 속성을 이용하여 처리하여야 함.

n. Ellipsis 처리 된 요소가 있을 때에는 마크업 파일 내에서 … 등을 사용해 임의로 줄이는 것이 아닌, text-overflow: ellipsis; 속성을 이용하여 처리.

o. 팝업 형태와 같이 height가 고정되어야 하는 형태가 아니라면, body 등의 요소의 height는 강제하지 않도록 구현.

p. 클릭 영역은 요소와, 그 주변 여백 영역을 포함한 범위로 지정되어야 함.

공통/컨텐츠 컴포넌트 관련 가이드

a. Common Component 피그마 섹션 내에 있는 요소는 common-component.html 파일 내에 우선적으로 구현 한 뒤, 페이지 마크업 파일 내에 동일한 내용을 참조하여 작성될 수 있도록 구성.

b. 컴포넌트 스타일의 경우 작성된 scss 파일을 참조하여 적용할 수 있도록 하여야 함.

c. 페이지 별 scss 파일 내에서 임의의 스타일을 적용하여, 공통 컴포넌트를 임의로 수정하는 행위를 금함.

d. 컨텐츠 컴포넌트의 경우 각 에픽 별 생성되는 별도의 컨텐츠 컴포넌트 섹션을 참고하여, 공통 컴포넌트와 같은 요령으로 구현 후 활용
